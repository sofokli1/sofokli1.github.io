Privacy Policy for Smoothly

Last updated: September 15, 2025

Introduction

Welcome to Smoothly (formerly SmoothieMaker), an iOS app that generates personalized smoothie recipes using AI and computer vision. This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use our mobile application.

By using Smoothly, you agree to the collection and use of information in accordance with this policy. If you do not agree with our practices, please do not use our app.

Information We Collect

Camera and Image Data

  • Purpose: We use your device's camera to identify fruits and vegetables for smoothie recipe generation
  • Processing: Images are processed locally on your device using Apple's Vision framework and Core ML models
  • Storage: Camera images are not stored, cached, or transmitted to our servers or any third parties
  • Permission: Camera access is optional - you can generate recipes without using the camera by manually selecting ingredients
  • Control: You can revoke camera permissions at any time in iOS Settings > Privacy & Security > Camera > Smoothly

Recipe and Usage Data

  • Recipe Information: Ingredients selected, generated recipes, recipe preferences, and nutritional information
  • Usage Tracking: Weekly recipe generation count (stored locally for free tier limits)
  • App Settings: Theme preferences, subscription status, and user preferences
  • Storage: All data is stored locally on your device using SwiftData - no cloud backup, sync, or server storage

Analytics Data

  • Status: All analytics collection is completely disabled via Firebase configuration
  • No Tracking: We do not collect any usage statistics, behavioral data, performance metrics, or user identifiers
  • Privacy First: Your app usage remains completely private - all analytics events are only logged locally for debugging
  • Technical Implementation: FIREBASE_ANALYTICS_COLLECTION_DEACTIVATED is set to true in app configuration

Subscription Data

  • Provider: RevenueCat handles all subscription processing and payment transactions
  • Data Collected: Subscription status, purchase history, entitlements, and anonymous customer identifiers
  • Purpose: Managing premium features, enforcing usage limits for free users, and processing subscription renewals
  • Third-Party Processing: RevenueCat processes subscription data according to their privacy policy

Technical Data

  • Remote Configuration: App feature flags and settings (weekly limits, ML confidence thresholds) fetched from Firebase Remote Config
  • No Personal Data: Remote Config requests contain no personal information or device identifiers
  • Error Logs: Local error logs for debugging only - not transmitted to servers
  • No Device Tracking: We do not collect device identifiers, UDID, advertising IDs, or technical specifications
  • Machine Learning: CoreML models run entirely on-device with no data transmission

How We Use Your Information

Core App Functionality (Local Only)

  • Process camera images locally using Vision framework and CoreML models (images never leave your device)
  • Generate personalized smoothie recipes using OpenAI's GPT-4o-mini model (only selected ingredients and dietary preferences sent)
  • Store your recipe history, preferences, and usage data locally using SwiftData
  • Manage subscription status and premium features through RevenueCat
  • Enforce weekly usage limits for free users (3 recipes per week, configurable via Remote Config)

Service Improvement

  • No analytics or tracking - we've removed all data collection
  • Remote configuration updates for app settings (no personal data)
  • Recipe generation improvements based on OpenAI's general improvements

Communication

  • Send important updates about the app (if subscribed to notifications)
  • Respond to support requests

Third-Party Services

OpenAI

  • Purpose: Generate smoothie recipes based on selected ingredients using GPT-4o-mini model
  • Data Shared: Only ingredient names and basic dietary preferences (no personal information)
  • Privacy: No camera images, device identifiers, or personal data are shared with OpenAI
  • API Usage: Requests are made directly to OpenAI's API with obfuscated API keys for security
  • Policy: OpenAI Privacy Policy

Firebase (Google) - Limited Use

  • Services: Remote Config only (Analytics, Firestore, and Authentication have been removed)
  • Data: No personal data transmitted - app only fetches configuration values (weekly limits, ML thresholds)
  • Purpose: Dynamic app configuration and feature flags
  • Privacy: All analytics collection is disabled via FIREBASE_ANALYTICS_COLLECTION_DEACTIVATED
  • Policy: Google Privacy Policy

RevenueCat

  • Purpose: Subscription management, payment processing, and premium feature access
  • Data: Purchase history, subscription status, anonymous customer identifiers, and App Store receipt data
  • Integration: Manages weekly recipe limits for free users and unlimited access for premium subscribers
  • Policy: RevenueCat Privacy Policy

Data Storage and Security

Local Storage

  • All user data is stored locally using SwiftData (Apple's secure local database framework)
  • Recipes, preferences, and usage tracking are stored securely on your device with iOS data protection
  • Data is sandboxed and not accessible to other apps
  • All local data is automatically deleted when you uninstall the app

No Cloud Storage

  • The app does not use any cloud storage services
  • No data is backed up to iCloud, Firebase, or any remote servers
  • Your recipe history and preferences remain entirely on your device

Security Measures

  • All network communications use HTTPS encryption with certificate pinning
  • API keys are obfuscated and securely stored using multiple layers of protection
  • Camera processing uses Apple's Vision framework with on-device CoreML models only
  • No sensitive data is logged, cached, or transmitted unnecessarily
  • Timeout protections prevent hanging network requests
  • Input validation ensures only safe ingredients are processed

Your Privacy Rights

GDPR Rights (EU Users)

  • Access: Request a copy of your personal data
  • Rectification: Correct inaccurate personal data
  • Erasure: Request deletion of your personal data
  • Portability: Receive your data in a portable format
  • Objection: Object to processing of your personal data
  • Consent Withdrawal: Withdraw consent for analytics at any time

California Privacy Rights (CCPA)

  • Right to Know: What personal information we collect and how it's used
  • Right to Delete: Request deletion of your personal information
  • Right to Non-Discrimination: Equal service regardless of privacy choices

How to Exercise Your Rights

  • Local Data: All data is stored locally - simply delete the app to remove all personal information
  • Email: Contact us at privacy-smoothly@juxhinbakalli.com for any privacy-related requests
  • Subscription Data: Contact RevenueCat support for subscription-related data requests
  • No Account Required: The app doesn't use accounts, so no account deletion process is needed

Children's Privacy

Smoothly is not intended for children under 13. We do not knowingly collect personal information from children under 13. If we discover that a child under 13 has provided us with personal information, we will delete it immediately.

Data Retention

Local Data

  • Recipe history, preferences, and usage data are stored locally until you delete the app
  • No data is retained on external servers - all information is device-only
  • SwiftData automatically manages local storage and cleanup

Data Deletion

  • Since all data is stored locally, simply deleting the app removes all your personal information
  • For subscription data managed by RevenueCat, contact privacy-smoothly@juxhinbakalli.com
  • Third-party services (OpenAI, RevenueCat) retain data according to their respective policies

International Data Transfers

Since most data processing occurs locally on your device, international data transfers are minimal:

  • OpenAI API requests (ingredient lists only) are processed in the United States
  • Firebase Remote Config requests (no personal data) may be processed globally by Google
  • RevenueCat subscription data is processed according to their global infrastructure
  • All camera images and recipe data remain on your device only

Changes to This Privacy Policy

We may update this Privacy Policy from time to time. Changes will be posted in the app and on our website. Continued use of the app after changes constitutes acceptance of the new policy.

  • Notification: In-app notification for material changes
  • Effective Date: Changes take effect immediately upon posting
  • Version History: Previous versions available upon request
